### 2.2 与其他模块的接口设计

#### 2.2.1 上游接口（输入）

| 接口来源 | 数据类型 | 数据内容 | 接口方式 |
|---------|---------|---------|---------|
| **插图生成模块** | 图片文件 | 故事分镜插图序列 | OSS URL |
| **故事模块** | 文本数据 | 故事文本、分段信息、情感标签 | 数据库查询 |
| **任务模块** | 任务配置 | 任务ID、生成参数、优先级 | 消息队列 |

#### 2.2.2 下游接口（输出）

| 接口目标 | 数据类型 | 数据内容 | 接口方式 |
|---------|---------|---------|---------|
| **音频合成模块** | 视频文件 | 无声视频片段序列 | OSS URL |
| **作品模块** | 元数据 | 视频时长、分辨率、格式 | 数据库写入 |
| **前端展示** | 进度状态 | 生成进度、预览URL | WebSocket推送 |

#### 2.2.3 接口数据结构定义

```python
# 视频生成任务输入
class VideoGenerationInput:
    task_id: str                    # 任务唯一标识
    project_id: str                 # 项目ID
    story_id: str                   # 故事ID
    story_text: str                 # 故事文本（作为提示词）
    segments: List[StorySegment]    # 故事分段列表
    images: List[ImageInfo]         # 插图序列
    
class StorySegment:
    segment_id: str                 # 分段ID
    text: str                       # 分段文本
    duration: float                 # 预估时长(秒)
    emotion: str                    # 情感标签
    scene_description: str          # 场景描述
    
class ImageInfo:
    image_id: str                   # 图片ID
    image_url: str                  # 图片OSS地址
    resolution: str                 # 分辨率
    segment_id: str                 # 关联的分段ID

# 视频生成任务输出
class VideoGenerationOutput:
    task_id: str                    # 任务ID
    status: str                     # 状态
    progress: int                   # 进度百分比
    videos: List[VideoSegment]      # 生成的视频片段
    error_message: str              # 错误信息
    
class VideoSegment:
    video_id: str                   # 视频ID
    video_url: str                  # 视频OSS地址
    duration: float                 # 时长(秒)
    resolution: str                 # 分辨率
    fps: int                        # 帧率
    file_size: int                  # 文件大小(字节)
    segment_id: str                 # 关联的分段ID
```

### 3.2 图片预处理技术细节

| 处理步骤 | 技术方案 | 说明 |
|---------|---------|------|
| 格式转换 | Pillow/OpenCV | 统一转换为RGB格式 |
| 尺寸调整 | Lanczos插值 | 保持16:9比例，最小1280×720 |
| 色彩校正 | 自动白平衡 | 确保色彩一致性 |
| 压缩优化 | WebP/高质量JPEG | 减少传输带宽 |
| 特征提取 | CLIP/ViT | 生成图片embedding |

### 3.3 图片质量要求

| 参数 | 要求 | 说明 |
|-----|------|------|
| 分辨率 | ≥1280×720 | 适配视频输出 |
| 宽高比 | 16:9 | 标准视频比例 |
| 格式 | JPG/PNG | 支持透明通道 |
| 色彩空间 | sRGB | 标准色彩空间 |
| 文件大小 | ≤5MB | 单张图片限制 |

## 5. AI视频生成核心技术选型与实现思路

### 5.1 技术选型对比分析

| 技术方案 | 优势 | 劣势 | 适用场景 | 推荐度 |
|---------|------|------|---------|--------|
| **Runway Gen-2** | 效果好、API成熟 | 成本高、有区域限制 | 高质量商业项目 | ★★★★☆ |
| **Pika Labs** | 性价比高、易集成 | 效果略逊 | 中等质量需求 | ★★★★☆ |
| **Sora (OpenAI)** | 效果最佳 | 未开放API、成本未知 | 未来规划 | ★★★☆☆ |
| **Stable Video Diffusion** | 开源免费、可私有部署 | 需要GPU资源、效果一般 | 成本敏感场景 | ★★★☆☆ |
| **自研模型** | 完全可控 | 研发成本高、周期长 | 长期战略 | ★★☆☆☆ |

### 5.4 核心处理流程

```
视频生成核心流程:

1. 任务接收
   ├── 从消息队列获取任务
   ├── 验证任务参数
   └── 初始化任务状态

2. 资源准备
   ├── 下载源图片
   ├── 生成优化提示词
   ├── 准备AI模型参数
   └── 加载背景音乐配置(音量/循环/淡入淡出)

3. AI生成调用
   ├── 选择AI服务商(Runway/Pika)
   ├── 调用图生视频API
   ├── 轮询生成状态
   └── 获取生成结果

4. 后处理
   ├── 视频格式转换
   ├── 添加Ken Burns效果
   ├── 生成缩略图
   └── 上传到OSS

5. 背景音乐叠加
   ├── 加载背景音乐文件
   ├── 音量调节(百分比→分贝映射)
   ├── 淡入淡出处理
   ├── 循环拼接(crossfade交叉过渡)
   ├── Ducking语音避让处理

│         │                                                                   │
│         ▼                                                                   │
│  ┌──────────────┐                                                           │
│  │  年龄段匹配   │ ← 年龄: 0-3/3-6/6-9/9-12                                │
│  └──────┬───────┘                                                           │
│         │                                                                   │
│         ▼                                                                   │
│  ┌──────────────┐                                                           │
│  │  综合评分排序 │ ← 权重: 情感40% + 分类30% + 年龄20% + 热度10%           │
│  └──────┬───────┘                                                           │
│         │                                                                   │
│         ▼                                                                   │
│  ┌──────────────┐                                                           │
│  │  推荐列表输出 │ ← Top 10 候选音乐                                        │
│  └──────────────┘                                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**推荐算法权重计算公式:**

```
Score(music) = w1 × emotion_match + w2 × category_match + w3 × age_match + w4 × popularity

其中:
  w1 = 0.4  (情感匹配权重)
  w2 = 0.3  (分类匹配权重)
  w3 = 0.2  (年龄段匹配权重)
  w4 = 0.1  (热度权重)

emotion_match: 音乐情感标签与故事情感标签的余弦相似度
category_match: 音乐风格与故事分类的匹配度 (0或1)
age_match: 音乐适用年龄与目标年龄的重叠度
popularity: 播放次数归一化值 (log(play_count) / log(max_play_count))
```

**情感-音乐风格映射表:**

| 故事情感 | 推荐音乐风格 | BPM范围 | 示例 |
|---------|------------|---------|------|
| 温馨 | 轻柔钢琴、摇篮曲 | 60-80 | "月光摇篮曲" |
| 欢快 | 活泼木琴、尤克里里 | 100-130 | "快乐小精灵" |
| 紧张 | 弦乐拨奏、低音鼓 | 80-110 | "森林冒险" |
| 神秘 | 竖琴、空灵合成器 | 70-90 | "魔法仙境" |
| 悲伤 | 缓慢弦乐、钢琴 | 50-70 | "离别之歌" |

#### 10.6.2 背景音乐混音处理示例

```python
import subprocess
import tempfile
import os

def mix_bgm_to_video(
    video_path: str,
    bgm_path: str,
    output_path: str,
    volume: int = 30,
    fade_in: float = 1.0,
    fade_out: float = 1.5,
    loop: bool = True,
    start_offset: float = 0.0,
) -> str:
    """
    将背景音乐混入视频文件

    参数:
        video_path: 输入视频文件路径
        bgm_path: 背景音乐文件路径
        output_path: 输出视频文件路径
        volume: 背景音乐音量百分比(0-100)
        fade_in: 淡入时长(秒)
        fade_out: 淡出时长(秒)
        loop: 是否循环播放
        start_offset: 音乐起始偏移(秒)

    返回:
        output_path: 输出文件路径

    异常:
        subprocess.CalledProcessError: FFmpeg执行失败
        FileNotFoundError: 输入文件不存在
    """
    if not os.path.exists(video_path):
        raise FileNotFoundError(f"视频文件不存在: {video_path}")
    if not os.path.exists(bgm_path):
        raise FileNotFoundError(f"音乐文件不存在: {bgm_path}")

    volume_factor = volume / 100.0

    video_duration = float(subprocess.check_output([
        "ffprobe", "-v", "error",
        "-show_entries", "format=duration",
        "-of", "default=noprint_wrappers=1:nokey=1",
        video_path
    ]).decode().strip())

    bgm_filter_parts = []

    if start_offset > 0:
        bgm_filter_parts.append(f"atrim=start={start_offset}")

    if loop:
        bgm_filter_parts.append("aloop=loop=-1:size=2e+09")

    bgm_filter_parts.append(f"volume={volume_factor}")

    if fade_in > 0:
        bgm_filter_parts.append(f"afade=t=in:st=0:d={fade_in}")

    if fade_out > 0:
        fade_out_start = max(0, video_duration - fade_out)
        bgm_filter_parts.append(
            f"afade=t=out:st={fade_out_start}:d={fade_out}"
        )

    bgm_filter = ",".join(bgm_filter_parts)

    cmd = [
        "ffmpeg", "-y",
        "-i", video_path,
        "-i", bgm_path,
        "-filter_complex",
        f"[1:a]{bgm_filter}[bgm];[0:a][bgm]amix=inputs=2:duration=first:dropout_transition=2[aout]",
        "-map", "0:v",
        "-map", "[aout]",
        "-c:v", "copy",
        "-c:a", "aac",
        "-b:a", "192k",
        "-shortest",
        output_path
    ]

    subprocess.run(cmd, check=True, capture_output=True)

    return output_path
```

#### 10.6.3 Ducking（语音避让）处理示例

```python
def apply_ducking(
    voice_path: str,
    bgm_path: str,
    output_path: str,
    bgm_volume: int = 30,
    duck_threshold: float = -30.0,
    duck_amount: float = 6.0,
    attack: float = 0.3,
    release: float = 0.5,
) -> str:
    """
    应用Ducking效果: 当语音活跃时自动降低背景音乐音量

    参数:
        voice_path: 语音音频文件路径
        bgm_path: 背景音乐文件路径
        output_path: 输出文件路径
        bgm_volume: 背景音乐基础音量百分比
        duck_threshold: 语音检测阈值(dB)
        duck_amount: 避让时降低的音量(dB)
        attack: 攻击时间(秒)
        release: 释放时间(秒)

    返回:
        output_path: 输出文件路径
    """
    volume_factor = bgm_volume / 100.0

    cmd = [
        "ffmpeg", "-y",
        "-i", voice_path,
        "-i", bgm_path,
        "-filter_complex",
        (
            f"[0:a]aresample=44100,pan=mono|c0=c0,"
            f"highpass=f=200,lowpass=f=3000,"
            f"compand=.3|.3:1|1:-90/-60|-60/-40|-40/-30|-30/-20|-20/-6|-6/0:0:0"
            f"[voice];"
            f"[1:a]aresample=44100,volume={volume_factor},"
            f"aloop=loop=-1:size=2e+09,"
            f"sidechaincompress=threshold={duck_threshold}dB:"
            f"ratio=4:attack={attack}:release={release}:makeup={duck_amount}dB"
            f"[bgm];"
            f"[voice][bgm]amix=inputs=2:duration=first:dropout_transition=2[aout]"
        ),
        "-map", "[aout]",
        "-c:a", "aac",
        "-b:a", "192k",
        output_path
    ]

    subprocess.run(cmd, check=True, capture_output=True)
    return output_path
```
