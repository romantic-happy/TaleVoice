## 项目基本信息

### 一、项目主题

**讲童话故事的软件**：根据故事文本生成高质量的语音故事。利用智能体的执行能力，综合使用多个软件，将故事写作、语音生成、配音、文生图、剪辑等串联成故事生产线，用你的声音对故事进行个性化。可以用伊索语言做测试。 

### 二、项目名称

童话之声（TaleVoice）

### 三、团队协作规则

- **每周例会时间**：每周日 18:00
- **Daily Scrum时间**：每天 20:00
- **沟通渠道**：飞书群
- **任务完成标准**：代码通过本地测试 + 提交PR + 至少一人Review
- **文档提交规范**：使用Markdown，统一存放在项目 `docs/` 目录
- **冲突沟通流程**：私下沟通 -> PM协调 -> 全员投票
- **分支管理**：
  - `main`：主分支，用于发布稳定版本
  - `develop`：开发分支，集成所有功能开发
  - `feature/*`：功能分支，用于开发新功能
  - `bugfix/*`：修复分支，用于修复bug
- **分支管理规范**：
  - 所有功能开发都在`feature/*`分支上进行
  - 功能开发完成后，合并到`develop`分支
  - 发布前，将`develop`分支合并到`main`分支
  - 修复bug时，在`bugfix/*`分支上进行，修复完成后合并到`develop`和`main`分支
